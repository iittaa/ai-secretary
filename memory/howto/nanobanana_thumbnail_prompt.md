# Gemini nano banana サムネ生成プロンプト【ベース】

背景画像＋キャラ画像＋テキストで特典サムネを作るときの**ベースプロンプト**。
今後nanobananaでサムネ生成するときは、これを起点に調整する。

## 使い方
- 画像1に背景、画像2にキャラを添付
- `[ ]` と `○○` を実際の値に置換
- 文字化けしたら文言を短く or Canvaで後乗せ

## プロンプト本体

```
画像1を背景、画像2をキャラクターとして使用し、特典告知用サムネイル画像を生成してください。

【合成指示】
- 背景：画像1をそのまま使用。改変・再描画しない
- キャラクター：画像2の人物を右下エリアに配置（ただし画面の端には絶対に接しないこと）
- キャラの顔・髪型・服装・画風は画像2と完全に同一に保つ（再描画禁止）
- キャラのサイズは画面の約10%
- 【厳守】キャラの右端と画面右端の間に、画面幅の10%以上の余白を必ず確保すること
- 【厳守】キャラの下端と画面下端の間に、画面高さの10%以上の余白を必ず確保すること
- 言い換えると、キャラのバウンディングボックスは画面の「右端から10%〜20%の帯」かつ「下端から10%〜20%の帯」の内側に収まる位置に配置する
- キャラが画面の端や隅にトリミングされたり、画面外にはみ出したりしてはいけない
- 背景とキャラの境界は自然に馴染ませる（軽い影を足してOK）

【テキスト挿入】
- テキスト配置エリア：キャラクター領域を除いた画面左側〜中央エリア（画面左端〜画面幅の約75%まで）

【揃えとサイズの基本ルール】
- 横揃えは必ず「左揃え（left-aligned）」。中央揃え・右揃えは禁止
- 1行目の左端と2行目の左端のX座標は完全に一致させること（縦のラインが揃って見えること）
- サイズは各行が配置エリア幅の80%以内に収まるよう、行ごとに自動調整してよい
- ただし1行目と2行目のサイズ差は最大でも1.5倍以内に収めること（極端なタイトル/サブタイトル差は避ける）
- 書体・太さ・文字色・フチ・影などデザインは1行目・2行目で完全に統一すること
- 英語補足：Both lines must be left-aligned and share the same left x-coordinate. Size may differ per line but within a 1.5x ratio. Typeface, weight, color, outline, and shadow must be identical across lines.

- メインテキスト（鉤括弧「」は文言の区切り表記のみ。実際の画像には鉤括弧・引用符・カッコ類を一切描画しないこと）：
  - 1行目の文言：{{LINE1}}  ← ここを実際の文言に置き換えて使用（例：月収30万円）
  - 2行目の文言：{{LINE2}}  ← ここを実際の文言に置き換えて使用（例：マネタイズロードマップ）
  - 描画される文字は上記の文言そのもののみ。「」『』""''（）[]などの記号は絶対に含めないこと
  - 位置：配置エリアの上下中央・左揃え（画面左端から画面幅の約5〜7%の位置に両行の左端を揃える）
  - 改行：上記の通り2行に分けて表示
  - スタイル：高級感のある明朝体またはセリフ書体（上品で太すぎない）、
    文字色はシャンパンゴールド〜ディープゴールドのグラデーション、
    濃紺または黒のごく細いフチ、淡いドロップシャドウで立体感を付与
- 全体トーン：派手さより上質さ・高級感を優先（ネオンや原色は使わない）

【重要：重なり禁止ルール】
- キャラクターとテキスト（メイン・サブ共）は一切重ならないこと
- キャラの外形（髪・体・装飾含む）とテキストのバウンディングボックスの間に最低でも画面幅の約3%の余白を確保
- 重なりそうな場合は、テキストサイズを縮小するか、テキスト配置エリアを左にずらして回避すること
- 日本語は一字一句そのまま正確に描画すること（文字化け厳禁）

【出力】
- アスペクト比：16:9
- 高コントラスト、視認性重視、クリックしたくなる派手さ
```

## コツ
- キャラが別人化したら「画像2のキャラクター性を100%維持、顔を再描画しない」を追加
- 配置・サイズ・色は具体的な数値/単語で指定すると精度上がる
- 英語版プロンプトの方が安定することも多い（日本語テキスト部分だけ「Japanese text, render exactly as:」で挟む）
